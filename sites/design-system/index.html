Perpetuals Design System

Atomic Design Methodology

This design system follows Brad Frost's Atomic Design methodology, organizing components into five distinct levels of complexity and reusability.

Atoms

The fundamental building blocks. Colors, typography, icons, buttons, and form inputs that can't be broken down further.

16 components

Molecules

Simple groups of atoms functioning together. Form fields, cards, search bars, and button groups.

12 components

Organisms

Complex UI sections composed of molecules and atoms. Headers, footers, hero sections, and feature grids.

8 components

Templates

Page-level structures that arrange organisms. Homepage, features page, and article layouts.

3 templates

Design Principles

The foundational guidelines that shape every component and decision in our design system.

01

Clarity Over Cleverness

Every element should have a clear purpose. We prioritize intuitive interfaces over complex interactions that require learning.

02

Consistency Breeds Trust

Consistent patterns and behaviors create predictability. Users should feel confident in their interactions across all touchpoints.

03

Performance is a Feature

Speed and responsiveness are not afterthoughts. Every component is optimized for performance from the start.

04

Accessibility by Default

Inclusive design is not optional. Every component meets WCAG 2.1 AA standards and works with assistive technologies.

05

Progressive Enhancement

Core functionality works everywhere. Enhanced experiences layer on top for modern browsers and devices.

06

Composable by Nature

Components are designed to work together. Mix and match atoms to create molecules, organisms, and beyond.

Icons

SVG icons designed at 24x24 with 2px strokes for clarity and consistency.

Icon Guidelines

  • All icons are SVG-based for crisp rendering at any size
  • Default size: 24x24px with 2px stroke width
  • Use currentColor for stroke to inherit text color
  • Icons should be wrapped in accessible containers with aria-labels
