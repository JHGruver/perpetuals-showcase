Intentional Friction & Trust Design

Modern fintech faces a fundamental tension: users want frictionless experiences, but security requires appropriate friction, and regulations mandate certain friction points.

The Friction Spectrum

Remove Friction

  • Browse products
  • View portfolio
  • Read research
  • Price alerts

Balance

  • Authentication
  • Order confirmation
  • Risk warnings
  • 2FA challenges

Add Friction

  • Large transfers
  • Account changes
  • Withdrawal
  • Password reset

Trust-Building Patterns

Visible Security

  • Lock icons near sensitive fields
  • HTTPS/Security badges
  • Compliance logos (MiFID, SEC)
  • Insurance mentions

Confirmation Layers

  • Select action (Buy/Sell)
  • Enter amount
  • Review order summary
  • Confirm (swipe/hold/PIN)
  • Confirmation + Receipt

Risk Communication

  • Appear at decision point
  • Clear, non-technical language
  • Actionable context
  • Reserve for genuine risks

Process Transparency

  • Explain KYC requirements
  • Show transaction times
  • Display fees before confirm
  • Clarify data usage

Key Requirements

Visual

  • 4.5:1 contrast ratio minimum
  • Support 200% zoom
  • Alt text for all images
  • Don't rely solely on color

Motor

  • Full keyboard navigation
  • 44x44px minimum touch targets
  • Adjustable time limits
  • Confirmation for irreversible actions

Cognitive

  • Consistent navigation location
  • Clear, descriptive labels
  • Specific error messages
  • Contextual help available
