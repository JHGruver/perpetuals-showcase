User Personas

Four primary user archetypes that shape every design decision in APPXISH.

👤
Alex Chen
Retail Beginner
Age: 24 — Recent college grad, crypto-curious
Goals: First derivatives trade, understand leverage without losing everything
Pain points: Overwhelmed by complex UIs, afraid of liquidation, confused by funding rates
Tech comfort: High (apps), Low (finance)
Key tasks: Simple buy/sell, check P&L, withdraw gains
Entry point: Barrier Futures (no margin complexity) via Upside Only brand
📈
Sarah Miller
Active Trader
Age: 32 — 5+ years trading experience, migrating from Binance
Goals: Better execution, lower fees, regulated platform
Pain points: Slow order fills, lack of advanced order types, regulatory uncertainty
Tech comfort: High (both apps and finance)
Key tasks: Multi-order management, chart analysis, rapid execution
Entry point: Full trading panel with all order types, keyboard shortcuts
🏢
David Park
Professional / Institutional
Age: 41 — Prop desk trader, manages $50M+ portfolio
Goals: API access, FIX protocol, prime brokerage services
Pain points: Counterparty risk, slow settlement, compliance requirements
Tech comfort: Expert (builds custom trading systems)
Key tasks: Large block trades, cross-margin, portfolio risk monitoring
Entry point: Desktop Perpetuals platform (mobile = monitoring only)
💼
Maria Santos
Broker / Introducing Broker
Age: 38 — Runs a network of 200+ retail clients
Goals: Client management dashboard, commission tracking, white-label options
Pain points: Fragmented platforms, manual reporting, client onboarding friction
Tech comfort: Moderate (business tools, not engineering)
Key tasks: Monitor client activity, manage sub-accounts, generate reports
Entry point: Broker portal (separate from main trading UI)

Prototype Notes

Interaction patterns, micro-interactions, gesture support, and animation specifications for the APPXISH prototype.

Button States

Default: Full opacity, standard background

Pressed: Scale 0.97, opacity 0.8 (100ms ease-out)

Disabled: 40% opacity, no pointer events

Loading: Pulse animation, spinner replaces text

Screen Transitions

Tab switch: Crossfade, 200ms ease

Push navigation: Slide left, 300ms cubic-bezier(0.16,1,0.3,1)

Modal open: Slide up from bottom + backdrop fade, 350ms

Modal close: Slide down + backdrop fade, 250ms

Gesture Support

Swipe left/right: Switch between Chart ↔ Order Book ↔ Trade Feed

Pull-to-refresh: Portfolio and market data screens

Long press: Position card → quick actions (close, edit, add TP/SL)

Pinch-to-zoom: Chart time frame adjustment

Real-Time Updates

Price ticks: Number flash green/red on change (300ms)

P&L updates: Smooth number counter animation

Order book: Row slide-in/out for new/removed orders

Trade feed: New rows prepend with fade-in (150ms)

Loading States

Skeleton screens: Pulsing gray rectangles matching final layout

Chart loading: Skeleton with gradient shimmer

Data refresh: Subtle top-bar progress indicator

Order submission: Button → spinner → checkmark → dismiss

Animation Specs

Easing: cubic-bezier(0.16, 1, 0.3, 1) for primary

Micro: 100-200ms for UI feedback

Navigation: 250-350ms for screen transitions

Data: 300ms for number changes and chart updates

60fps minimum for all animations

Haptic Feedback

Order submitted: Medium impact

Order filled: Success notification

Barrier triggered: Heavy impact + warning

Slider snaps: Light impact at tick marks

Accessibility

VoiceOver: All interactive elements labeled

Dynamic Type: Supports iOS text scaling

Color contrast: WCAG AA minimum (4.5:1)

Reduced motion: Respects system preference
